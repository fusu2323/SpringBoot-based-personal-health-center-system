# 智能化升级方案：AI 功能接入规划

基于当前的健康管理系统（Spring Boot 3 + Vue 3），我们引入大语言模型（LLM）可以将系统从“被动记录”升级为“主动顾问”。以下是为您策划的 4 个可行性高、适合毕业设计的 AI 功能模块。

## 1. 核心推荐功能

### 🌟 功能一：AI 个人健康顾问 (智能对话助手)
**推荐指数**：⭐⭐⭐⭐⭐
**功能描述**：
在系统界面（如右下角）集成一个悬浮聊天窗口，用户可以像咨询医生或健身教练一样与 AI 对话。
**亮点设计 (Context-Aware)**：
AI 不仅是通用聊天，还能**读取用户的健康数据**。
*   **场景示例**：
    *   **用户**：“我最近感觉很累，减肥效果也不好。”
    *   **AI (后台已注入用户数据)**：“我看了一下您的记录，您最近一周平均每天睡眠不足 6 小时，且饮食中碳水占比过高（超过 60%）。建议您先保证 7 小时睡眠，并尝试将晚餐的主食减半。”
**技术实现**：
*   **后端**：集成大模型 API（如 DeepSeek、通义千问、OpenAI）。
*   **Prompt 工程**：在发送用户问题前，先查询该用户的最新身体指标（体重、BMI）、昨日运动/饮食数据，作为“背景信息”拼接在 System Prompt 中。
*   **前端**：实现流式输出（打字机效果），提升交互体验。

### 🌟 功能二：健康周报/日报智能生成
**推荐指数**：⭐⭐⭐⭐
**功能描述**：
用户登录首页时，展示一份“AI 健康分析简报”。
**功能细节**：
*   不再只是冷冰冰的图表，而是一段有温度的文字分析。
*   例如：“本周您的运动频率比上周提升了 20%，非常棒！但体重波动较大，可能是因为周末的两次高热量饮食（记录显示吃了火锅），建议下周注意控制欺骗餐的次数。”
**技术实现**：
*   后端触发：每天/每周定时或用户首次登录时，拉取过去一段时间的数据，发送给 LLM 生成一段 200 字左右的评价，存入数据库或 Redis。

## 2. 进阶/辅助功能

### 功能三：智能食谱与运动计划生成器
**推荐指数**：⭐⭐⭐
**功能描述**：
*   用户输入目标（如：“我想在 30 天内减重 5 斤”）。
*   AI 根据用户的当前体重、身高、年龄，生成一份详细的 **7 天膳食计划表** 和 **运动计划表**。
**技术实现**：
*   要求 LLM 返回标准的 JSON 格式数据，前端直接将其渲染为课程表/日历视图，用户可以一键保存到“计划表”中。

### 功能四：论坛内容智能审核 (内容安全)
**推荐指数**：⭐⭐⭐
**功能描述**：
*   在论坛发布帖子或评论时，自动识别辱骂、广告、色情或极端的医疗谣言。
*   识别出违规内容自动拦截或标记为“待人工审核”。
**技术实现**：
*   在 `savePost` 接口中加入 AOP 切面，调用 AI 的内容审核接口或使用 Prompt 让 LLM 判断情感倾向。

## 3. 技术落地架构

### 后端架构 (Spring Boot 3)
1.  **AI Client 层**：引入 `OkHttp` 或 `Spring AI` (Spring Boot 3 官方推荐) 来调用外部大模型 API。
2.  **Prompt Manager**：管理不同场景的提示词模板。
3.  **SSE (Server-Sent Events)**：用于实现对话的实时流式传输（比 WebSocket 更轻量，适合单向输出）。

### 前端架构 (Vue 3)
1.  **Markdown 渲染**：引入 `markdown-it` 或类似库，支持 AI 输出的加粗、列表格式。
2.  **流式响应处理**：处理 `fetch` 或 `EventSource` 的流数据，实现逐字显示。

## 4. 实施路线建议

为了保证项目进度，建议按以下顺序开发：

1.  **第一步 (基础对话)**：先跑通后端调用 LLM API 的流程，在前端做一个简单的对话框，能一问一答。
2.  **第二步 (数据注入)**：修改后端逻辑，在调用 AI 前先查询 `HealthIndicator` 表和 `SportRecord` 表，把数据塞给 AI，实现“懂你的 AI”。
3.  **第三步 (界面美化)**：仿照 ChatGPT/DeepSeek 的界面风格，做悬浮球和侧边栏。
