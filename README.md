# 健康管理系统（毕业设计）

一个基于前后端分离的健康管理平台，面向普通用户与管理员，支持健康指标、运动/饮食/睡眠记录、论坛互动与系统公告等功能，并提供数据可视化统计与富文本编辑体验。

## 项目概述
- 前后端分离，后端提供 RESTful API，前端使用现代化 Vue3 技术栈构建。
- 面向普通用户的日常健康数据记录与论坛互动；面向管理员的全量数据管理与系统公告维护。
- 数据统计与趋势展示，辅助用户形成健康习惯。

## 技术栈与环境要求
- 后端：`SpringBoot 3`、`MyBatis`、`Hutool`
- 前端：`Vue 3`、`Element Plus`、`Vue-Router`、`Axios`
- 数据库：`MySQL`
- 版本要求：
  - `JDK ≥ 17`
  - `MySQL 5.7 / 8.0`
  - `Node.js ≥ 18`
  - `Maven ≥ 3.8`
  - `Navicat ≥ 16`（推荐）

## 数据库设计概览
本项目采用 9 张核心业务表，满足用户记录与论坛交互的完整闭环：
- `用户表`：存储普通用户账户与基本资料
- `管理员表`：存储管理员账户与权限相关信息
- `公告表`：系统公告内容与展示控制
- `健康指标表`：身体指标（如体重、血压、心率等）每日记录
- `运动记录表`：运动类型、时长、消耗等每日记录
- `饮食记录表`：餐次、食物、能量摄入等每日记录
- `睡眠记录表`：睡眠时长、质量评分等每日记录
- `帖子表`：论坛帖子内容（支持富文本）
- `评论表`：帖子下的评论与回复

关系示意（简化）：
- 用户 1..N 记录（健康指标、运动、饮食、睡眠）
- 帖子 1..N 评论；评论可支持楼层回复（父子关系）
- 管理员维护公告与全量数据管理

## 系统架构与交互流程
- 架构：前端（`Vue3 + Element-Plus`）通过 `Axios` 调用后端（`SpringBoot3`）REST API；后端使用 `MyBatis` 访问 `MySQL`；通用工具与日期处理等由 `Hutool` 提供。
- 鉴权：登录成功后颁发会话凭证（如 Token/Session），前端路由守卫与接口拦截器统一处理。
- 可观察性：后端标准日志输出，必要的错误码与统一响应结构便于前端处理。

## 快速开始（本地运行）
以下为通用本地启动流程，具体以项目实际代码为准：

### 准备数据库
- 新建数据库：`healthy_db`
- 配置连接：`url`、`username`、`password`
- 导入初始化 SQL（如 `schema.sql` / `data.sql`），或使用后端的自动建表逻辑（若有）。

### 后端启动
- 本地环境：安装 `JDK 17+` 与 `Maven 3.8+`
- 配置 `application.yml`（示例，仅供参考）：
  ```yaml
  spring:
    datasource:
      url: jdbc:mysql://localhost:3306/healthy_db?useSSL=false&serverTimezone=UTC&characterEncoding=utf8
      username: root
      password: your_password
    jackson:
      time-zone: Asia/Shanghai
  mybatis:
    mapper-locations: classpath*:mapper/*.xml
  server:
    port: 8080
  ```
- 构建与运行：
  - `mvn clean package -DskipTests`
  - `java -jar target/healthy-*.jar`
  - 或直接：`mvn spring-boot:run`

### 前端启动
- 本地环境：安装 `Node.js 18+`
- 安装依赖：`npm install`
- 开发运行：`npm run dev`（默认端口如 `http://localhost:5173`）
- 生产构建：`npm run build`
- 配置后端地址：前端 `.env` / 配置文件中设置 `VITE_API_BASE = http://localhost:8080/api`

## 典型页面与功能说明
- 系统首页：公告展示、健康提醒、近7/30日关键指标趋势图（`ECharts`）
- 指标/运动/饮食/睡眠记录页：支持查询、分页、创建、编辑、删除；表单校验与批量操作
- 论坛：帖子列表、详情页、评论与楼层回复；支持富文本编辑器发布内容
- 用户中心：头像昵称、基础资料维护、密码修改
- 管理后台：用户/管理员账号管理，帖子与评论的审核与处理，公告发布

## 创新点与亮点
- 数据可视化：`ECharts` 展示多维健康数据（趋势、分布、对比），提升洞察力
- 复杂数据关联：围绕用户的多表记录与论坛交互关系，统一查询与统计口径
- 富文本编辑：帖子与评论支持富文本（图片、表情、代码块等），提升社区互动质量
- 体验与一致性：`Element-Plus` 组件体系，统一交互与视觉规范；路由守卫与接口拦截器保证会话一致性

## 开发约定与代码规范
- 接口规范：统一响应结构（状态码、消息、数据）；分页参数和排序字段标准化
- 异常处理：后端统一异常捕获与错误码；前端统一提示
- 安全与合规：不在代码库中硬编码密钥；敏感信息使用环境变量配置
- 命名与风格：遵循各自技术栈的主流最佳实践（Java 命名规范、Vue 组合式 API 等）

## 常见问题
- 端口冲突：修改 `server.port` 或前端开发端口
- 数据库初始化失败：确认 `url`、`权限`、`字符集` 与 `时区` 设置正确
- 构建失败：检查 `JDK`、`Node`、`Maven` 版本是否满足要求
- 跨域问题：在后端开启 `CORS` 或通过前端代理解决
